# Security Policy

## Supported Versions

We take the security of NJMTech Blob Cron seriously. The following versions are currently supported with security updates:

| Version | Supported          |
| :------ | :----------------- |
| `0.1.x` | :white_check_mark: |

## Reporting a Vulnerability

We appreciate responsible disclosure of security vulnerabilities. If you discover a security issue within NJMTech Blob Cron, please follow these steps:

1.  **Do NOT open a public issue.** Publicly disclosing vulnerabilities can put the project and its users at risk.
2.  **Email us directly** at `security@njmtech.com` (replace with actual security contact if available, otherwise use a placeholder).
3.  **Provide detailed information** about the vulnerability, including:
    *   A clear and concise description of the vulnerability.
    *   Steps to reproduce the vulnerability.
    *   The potential impact of the vulnerability.
    *   Any suggested mitigations or fixes.
    *   Your contact information (optional, but helpful for further communication).

We will acknowledge your report within 48 hours and provide a more detailed response and estimated timeline for remediation as quickly as possible.

## Disclosure Policy

- We will work with you to understand and address the vulnerability.
- We will keep you informed of our progress.
- We will publicly acknowledge your responsible disclosure in our release notes or a dedicated security advisory, if you agree.

Thank you for helping to keep NJMTech Blob Cron secure!
